## 🚀 部署步骤

1. 访问 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 下载适合你操作系统的版本
3. 安装完成

### 第2步：打开项目

1. 打开微信开发者工具
2. 点击 **+** 新建项目
3. 选择 **导入项目**
4. 填写信息：
   - **项目路径** - 选择 `mini-app` 文件夹（注意：不是整个仓库，只选 mini-app）
   - **AppID** - `wxb18d375bf89cf958`
   - **项目名称** - 睡了么
   - **开发框架** - 选择 **标准建立**
5. 点击 **导入**

### 第3步：关联云开发环境

1. 打开开发者工具
2. 点击上方菜单栏 **云开发**
3. 选择 **导入云环境**
4. 确认关联到环境 ID：`cloudbase-d7gcjzhmd28e9244c`

### 第4步：上传云函数

1. 在开发者工具左侧找到 **云开发** 按钮
2. 点击进入云开发控制台
3. 左侧菜单 **云函数**

#### 云函数2：recordSleep

- 复制 `cloud-functions/recordSleep/index.js` 的内容

#### 云函数3：getSleepStats

- 复制 `cloud-functions/getSleepStats/index.js` 的内容

### 第5步：创建数据库集合

在云开发控制台左侧 **数据库** 中创建以下3个集合：

#### 集合1：users（用户信息）
```
集合名称：users
字段结构：
- _id（自动生成）
- _openid（用户openid）
- nickname（昵称，字符串）
- avatar（头像，字符串）
- createTime（创建时间）
- totalSleepDays（记录天数）
- avgSleep（平均睡眠时长）
- avgQuality（平均睡眠质量）
```

#### 集合2：sleepRecords（睡眠记录）
```
集合名称：sleepRecords
字段结构：
- _id（自动生成）
- _openid（用户openid）
- date（日期，格式：YYYY-MM-DD）
- bedtime（入睡时间，格式：HH:MM）
- wakeup（起床时间，格式：HH:MM）
- duration（睡眠时长，数字，单位：小时）
- quality（睡眠质量，1-5，数字）
- notes（备注，字符串）
- createTime（创建时间）
```

#### 集合3：settings（应用设置）
```
集合名称：settings
字段结构：
- _id（自动生成）
- version（版本号）
- updateTime（更新时间）
```

### 第6步：配置数据库权限

对于每个集合，设置以下权限：

**users 集合：**
```
读：仅创建者可读
写：仅创建者可写
```

**sleepRecords 集合：**
```
读：仅创建者可读
写：仅创建者可写
```

### 第7步：编译测试

1. 在开发者工具点击 **编译** 或按 Ctrl+B（Windows）/ Cmd+B（Mac）
2. 可以在右侧模拟器看到小程序运行效果
3. 点击 **记录** 页面，尝试记录一条睡眠数据
4. 检查是否成功保存到云数据库

### 第8步：提交审核（可选）

如果你想发布到微信应用市场：

1. 在开发者工具点击上方 **上传** 按钮
2. 填写版本号和描述
3. 上传代码
4. 在小程序后台 → **开发管理** → **版本管理** 中可以看到上传的版本
5. 点击 **提交审核**
6. 等待微信审核（通常1-3个工作日）

### 第9步：发布版本

1. 审核通过后，在版本管理页面点击 **发布**
2. 小程序就上线了！

## 🔍 常见问题

### Q1：上传云函数失败？

A：确保：
1. 环境ID正确
2. 云函数名称与代码中的 `exports.main` 对应
3. 没有语法错误
4. 网络连接正常

### Q2：数据库无法连接？

A：检查：
1. 云环境已正确初始化
2. 集合已创建
3. 权限设置正确（改为 "仅创建者" 或 "所有人可读写" 以测试）

### Q3：小程序无法调用云函数？

A：
1. 确保 `app.js` 中的环境ID正确
2. 云函数已成功上传部署
3. 开发者工具中勾选 "本地调试"

### Q4：如何在真机上测试？

A：
1. 在开发者工具右上角找到 **预览** 按钮
2. 扫描生成的二维码
3. 用你的微信打开预览

## 📱 真实上线流程

当完全开发完成后：

1. **本地测试** - 确保所有功能正常
2. **提交审核** - 在小程序后台上传版本
3. **审核中** - 等待微信官方审核（1-3天）
4. **审核通过** - 发布版本
5. **上线** - 用户可在微信搜一搜找到你的小程序

## 💡 开发建议

1. **版本控制** - 每次发布新版本时更新 `package.json` 中的版本号
2. **备份数据** - 定期在云开发控制台导出数据库备份
3. **监控日志** - 在云开发 → 日志中查看云函数执行情况
4. **性能优化** - 监控云函数的执行时间和内存使用
